# Codex Island 交互打磨设计

## 背景

当前 Codex Island 已经具备会话展示、hover 展开、拖动吸附和手动隐藏能力，但实际使用中还存在三类明显问题：

1. 鼠标在岛本体和展开面板之间切换时，展开状态会闪烁。
2. 拖动后侧边吸附规则不符合直觉，窗口会落到左右侧上边缘，视觉上像“侧边条被贴到了顶部”。
3. 拖动、吸附、展开/收起的过渡不够顺，缺少统一的收敛节奏。

本次设计只处理交互体验，不改动会话数据模型、不扩展新的产品功能。

## 目标

- 鼠标移到展开面板时，面板保持展开，不因离开岛本体而立刻收起。
- 拖动到屏幕中间时保持浮动，不强制套用左右侧吸附样式。
- 靠近左、右边缘时才切换到侧边吸附，并改成纵向布局。
- 左右侧详情面板从内侧展开，而不是继续向下展开。
- 所有布局切换和展开收起都有一致、细腻的过渡感。

## 非目标

- 不重做会话列表内容结构。
- 不增加设置项、主题项或动画配置项。
- 不改变后端会话状态同步逻辑。
- 不引入复杂的拖拽跟随预览层。

## 交互模型

### 1. 悬浮区域

岛本体和展开面板要被视为同一个悬浮区域。

规则如下：

- 鼠标进入岛本体，触发展开。
- 鼠标从岛本体进入展开面板，不得触发收起。
- 只有鼠标真正离开整个岛区域后，才进入收起延迟。
- 正在拖动、正在吸附结算时，不响应 hover 收起计时器。

收起延迟应短于当前实现，但保留一点余量，避免鼠标轻微抖动就反复开合。

### 2. 吸附状态

窗口增加一个明确的浮动态：

- `floating`：屏幕中间或未命中边缘吸附带时使用。
- `top`：靠近屏幕顶部时使用。
- `left`：靠近左边缘时使用。
- `right`：靠近右边缘时使用。

吸附判定不再只看“哪个边最近”，而是看是否进入对应边缘的吸附带。默认吸附带建议为 72px：

- 进入顶部吸附带时，允许 `top`。
- 进入左侧吸附带时，允许 `left`。
- 进入右侧吸附带时，允许 `right`。
- 如果没有命中任何吸附带，保持 `floating`。

这样可以避免窗口在屏幕中间区域被强行套用侧边样式。

### 3. 侧边布局

当 `left` 或 `right` 生效时，岛本体切换成纵向形态：

- 收起态从横条变为竖条。
- 呼吸灯沿纵向排列。
- 展开面板从内侧横向展开。

方向约定：

- `left`：岛固定在左侧，详情向右展开。
- `right`：岛固定在右侧，详情向左展开。

`top` 保持现有的横向布局。

`floating` 则保持独立的非贴边形态：

- 收起态沿用横向胶囊。
- 展开态默认向下展开，沿用当前最稳妥的面板展开方向。
- 位置由拖动后的落点决定，不再强行贴到任何屏幕边缘。

### 4. 动画节奏

动画要拆成“状态切换”和“视觉收敛”两层，避免互相抢控制权。

建议节奏：

- hover 展开：短延迟后开始展开，时长偏短。
- hover 收起：在离开整个区域后再延迟收起，时长略长于展开。
- 拖动开始：立即冻结 hover 计时器，先收拢岛态。
- 拖动结束：先计算吸附结果，再进入新的布局过渡。
- 吸附反馈：只做轻微的弹性回弹，不做夸张位移。

动画属性只使用布局相关的平滑过渡，例如 `transform`、`opacity`、`clip-path`、`border-radius`，不引入额外的复杂动画层。

## 状态流

建议把前端状态拆成三部分理解：

1. 交互状态：`hovered`、`expanded`、`dragging`、`settling`
2. 布局状态：`floating`、`top`、`left`、`right`
3. 视觉状态：是否在吸附脉冲中、是否显示展开面板

关键规则：

- `dragging` 时不允许 hover 直接改写展开状态。
- `settling` 时允许布局切换，但不允许再次进入新一轮拖动判定。
- 布局状态由拖动结算结果决定，展开状态由 hover 逻辑决定。

## 后端配合

后端仍负责：

- 计算窗口最终吸附边缘。
- 设置对应的窗口尺寸和位置。
- 在前端请求吸附后返回最终 edge。

这里的返回值需要能表达“未吸附”：

- 命中吸附带时，返回 `top`、`left` 或 `right`。
- 未命中吸附带时，返回 `none`，前端保留 `floating`。
- `floating` 不应被视为一个边缘，而应视为独立的锚点状态。

前端负责：

- 根据 edge 切换横向或纵向布局。
- 根据 `floating` 决定是否保留中间浮动样式。
- 根据 hover 状态控制展开/收起。

## 错误处理

- 如果拖动或吸附 API 在普通浏览器预览中不可用，维持当前 UI 预览，不报错。
- 如果吸附结果缺失，保持 `floating`，不要强制切换成侧边布局。
- 所有计时器在卸载和拖动开始时都必须清理，避免悬挂回调造成闪烁。

## 测试

前端测试应覆盖：

- 鼠标从岛本体移动到展开面板时，不会触发收起。
- 鼠标离开整个岛区域后，才在延迟结束后收起。
- 进入左右边缘吸附带时，布局切换为纵向。
- 未命中吸附带时，保留浮动布局。
- 左右侧展开方向分别向内侧展开。
- 拖动后吸附结算只发生一次，不会重复触发布局抖动。

后端测试应覆盖：

- 左右边缘吸附结果仍然正确。
- 侧边布局尺寸和位置与吸附边缘一致。
- `floating` 与边缘吸附之间的切换不会把窗口误放到左/右侧顶部。

## 验收标准

1. 鼠标在展开面板上停留时，面板保持展开。
2. 轻微移出岛本体不会立刻收起。
3. 拖到中间区域时，窗口保持浮动，不出现错误的侧边条样式。
4. 拖到左/右边缘时，窗口切换为纵向侧边布局。
5. 左右侧详情面板从内侧展开，动画连贯，不突兀。
6. 拖动、吸附、展开收起之间没有明显闪烁或状态打架。
